Organización jerárquica de protocolos de redes de acuerdo con los niveles del modelo OSI

Nivel 1: Capa Física

Descripción: La capa física es la más baja del modelo OSI. Es responsable de la transmisión de bits individuales de un nodo a otro.

Ejemplos: Los ejemplos de protocolos utilizados en la capa física incluyen Ethernet (IEEE 802.3), Wi-Fi (IEEE 802.11), Bluetooth (IEEE 802.15), y otros.

Nivel 2: Capa de Enlace de Datos

Descripción: La capa de enlace de datos se encarga de la entrega de datos de nodo a nodo. Su función principal es asegurar que la transferencia de datos sea libre de errores de un nodo a otro, sobre la capa física.

Ejemplos: Algunos ejemplos de protocolos de la capa de enlace de datos incluyen Ethernet (IEEE 802.3), Token Ring, Frame Relay, y otros.

Capa 3: Capa de Red

Descripción: La capa de red proporciona los medios funcionales y de procedimiento para transferir secuencias de datos de longitud variable de un host fuente a un host de destino a través de una o más redes, manteniendo las funciones de calidad de servicio.

Ejemplos: Algunos ejemplos de protocolos de la capa de red incluyen IP (Internet Protocol), ICMP (Internet Control Message Protocol), IGMP (Internet Group Management Protocol), IPsec (Internet Protocol Security), y otros.

Capa 4: Capa de Transporte

Descripción: La capa de transporte asegura la entrega completa y confiable de paquetes de datos. Proporciona mecanismos como el control de errores, el control de flujo y el control de congestión para realizar un seguimiento de los paquetes de datos, verificar errores y duplicaciones, y reenviar la información que falla en la entrega.

Ejemplos: Algunos ejemplos de protocolos de la capa de transporte incluyen TCP (Transmission Control Protocol), UDP (User Datagram Protocol), DCCP (Datagram Congestion Control Protocol), y otros.

Capa 5: Capa de Sesión

Descripción: La capa de sesión controla las conversaciones entre diferentes computadoras. Una sesión o conexión entre máquinas se establece, se gestiona y se termina en la capa 5.

Ejemplos: Algunos ejemplos de protocolos utilizados en la capa 5 incluyen el Protocolo de Sesión AppleTalk (ASP), el Protocolo de Flujo de Datos AppleTalk (ADSP), el Protocolo de Capa de Sesión OSI (X.225, ISO 8327), y otros.

Capa 6: Capa de Presentación

Descripción: La capa de presentación establece el contexto entre las entidades de la capa de aplicación, en el que las entidades de capas superiores pueden usar diferentes sintaxis y semánticas si el servicio de presentación proporciona un mapeo entre ellas.

Ejemplos: Algunos ejemplos de protocolos utilizados en la capa de presentación incluyen el Protocolo de Archivos Apple (AFP), la Arquitectura de Computación Independiente (ICA), el Protocolo de Presentación Ligero (LPP), y otros.

Capa 7: Capa de Aplicación

Descripción: La capa de aplicación proporciona una interfaz para enviar y recibir datos. Esta capa proporciona los protocolos y servicios que la aplicación necesita para acceder a esos recursos.

Ejemplos: Algunos ejemplos de protocolos de la capa de aplicación incluyen HTTP (Hypertext Transfer Protocol), FTP (File Transfer Protocol), SMTP (Simple Mail Transfer Protocol), y otros.
